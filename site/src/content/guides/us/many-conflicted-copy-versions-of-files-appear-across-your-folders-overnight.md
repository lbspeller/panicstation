# What to do if… many “conflicted copy” versions of files appear across your folders overnight (USA)

## Short answer
Isolate the affected device from the internet immediately to stop the duplication loop, then use a separate, secure device to inspect your cloud account’s version history or rollback tools.

## Do not do these things
- Don't bulk-delete the duplicate files until you have verified which version contains the most recent data.
- Don't move or rename the original folders right now, as this can break the "Restore" or "Version History" links used by cloud providers.
- Don't ignore "selective sync" or "files on-demand" errors, which often trigger these mass duplications during system updates.
- Don't plug in your primary backup drive if you see strange file extensions or ransom notes.
- Don't restart the sync process on the affected device until the root cause is identified.

## What to do now
1. **Isolate the affected hardware immediately.**
   - **Disconnect the device from all internet sources by enabling Airplane Mode or unplugging the Ethernet cable.**
   - **Manually "Quit" or "Force Stop" the sync client (OneDrive, Dropbox, Google Drive, or iCloud) via the Task Manager or Activity Monitor.**
   - **Ensure any other devices linked to the same account are also offline to prevent the conflict from spreading.**

2. **Identify the specific naming pattern of the duplicates.**
   - **Check if the duplicates include phrases like "Conflicted Copy," a specific device name (e.g., "Work-Laptop"), or a bracketed number (e.g., "File (2)").**
   - **Note if the files have new, unrecognizable extensions (e.g., .crypt, .locked, .[random]) or if "ReadMe" text files have appeared in every folder.**
   - **If you see signs of mass renaming or encrypted files that will not open, treat the situation as a potential ransomware incident and skip to step 7.**

3. **Secure the cloud account from a separate, clean device.**
   - **Log in to your cloud provider's web interface (not the app) using a device that is not exhibiting the issue.**
   - **Review "Recent Activity" or "Security Sessions" to look for logins from unfamiliar locations or devices.**
   - **Enable Multi-Factor Authentication (MFA) if it is not already active, but avoid a password reset until you are ready to re-authenticate every device, as this can trigger a fresh sync wave.**

4. **Respond to system-level conflict prompts.**
   - **On macOS/iCloud, look for a "Conflict Resolution" dialog box that asks you to choose which version to keep.**
   - **Select the option to "Keep Both" if you are unsure, which allows you to sort them offline later without losing data.**

5. **Utilize cloud-side restoration tools.**
   - **Locate the "Version History" for a few critical files to confirm when the last clean save occurred.**
   - **Use "OneDrive Restore," "Dropbox Rewind," or "Google Drive Activity" to roll back the entire library to a point in time before the duplicates appeared.**
   - **Perform this rollback on the web interface only while your local devices remain offline.**

6. **Verify local system integrity before reconnecting.**
   - **Run a full malware scan using a reputable, updated security suite on the isolated device.**
   - **Check for "Selective Sync" errors in the sync client settings; sometimes unchecking and re-checking a folder can resolve a persistent conflict loop.**

7. **Escalate for professional or legal assistance if an attack is suspected.**
   - **Contact your organization's IT department or Help Desk immediately if this is a work or school account.**
   - **Report suspected ransomware or cyber-extortion to the FBI Internet Crime Complaint Center (IC3) at ic3.gov.**
   - **Consult the CISA Ransomware Response Checklist for structured containment steps if you are managing an organizational network.**

## What can wait
- Manually sorting through hundreds of individual "conflicted" vs "original" files.
- Reinstalling the operating system or wiping the hard drive.
- Running third-party "duplicate finder" or "deduplication" software.
- Communicating the issue to external collaborators until the account is secured.

## Important reassurance
Mass file duplication is frequently caused by a "sync loop" where two devices attempt to update the same folder simultaneously, or a system update resets your sync preferences. This usually means your data is still present in multiple forms rather than being deleted. By staying offline, you prevent the cloud from "deciding" to keep the wrong version.

## Scope note
This guide provides first-step stabilization for digital sync conflicts and potential account compromises. It does not cover long-term data forensics, hardware repair, or complex server-side data recovery.

## Important note
This information is for general guidance and does not constitute technical, legal, or security advice. Digital environments vary significantly based on software versions and account types. If you suspect your data has been targeted by a cybercriminal, prioritize isolation and seek qualified cybersecurity assistance.
