# What to do if… a document says it is locked for editing but you cannot find where it is open (USA)

## Short answer
Choose **Read-Only** if it is offered, and save any visible work as a separate copy before troubleshooting. Then close other sessions and check whether anyone else is editing before you delete or change anything.

## Do not do these things
- Do not delete several files from the folder to see what works.
- Do not overwrite the locked original with a troubleshooting copy.
- Do not keep retrying while OneDrive or SharePoint is still syncing; it can make version handling harder.
- Do not force an unlock on a shared or workplace document until you have checked whether someone else is editing.
- Do not use a forced shutdown unless the device is unresponsive; give the app time to finish saving.
- Do not install random third-party unlocker tools on a work computer.

## What to do now
1. **Protect the work you can see.**
   - If the app offers **Read-Only**, open that version.
   - If you can see changes that need preserving, use **Save As** or **Save a Copy** and create a separate file such as `filename - safe copy`.
   - Save the copy somewhere local, outside the shared folder, and leave the locked original in place.

2. **Check for another session.**
   - Look on your other devices, browser tabs, Office web apps, remote desktop sessions, and virtual machines.
   - Close the document anywhere you find it open.
   - Give the app or service a few minutes to release the lock before trying again.

3. **Check whether another person is editing.**
   - For a shared file, ask whether anyone currently has the document open.
   - If another person is editing, wait for them to close it or work from your separate copy.
   - For a workplace file, contact your IT help desk if you cannot confirm who has it open.

4. **Close a background app.**
   - Save any other work first.
   - On Windows, close Word, Excel, or PowerPoint normally. If the matching app remains open, use **Task Manager** to end only that app.
   - If you are unsure what to end, restart the computer instead of experimenting.

5. **For a Word document on Windows, remove an owner file only when it is safe.**
   - Use this only if you have confirmed that nobody else is editing and all Word windows are closed.
   - In the same folder as the Word document, look for a small file whose name begins with `~$` and matches the document name.
   - Delete only that matching `~$` owner file, then reopen the Word document.
   - If you are unsure which file is the matching owner file, stop and ask IT.

6. **For OneDrive or SharePoint, check sync before changing the shared original.**
   - Check the OneDrive status icon and wait if syncing is still in progress.
   - If OneDrive syncing is paused, resume it and allow syncing to complete.
   - Do not move, rename, or delete the shared original while you are unsure which version is current.
   - If versions have diverged, use version history or ask IT before restoring, merging, or deleting anything.

7. **On macOS, close a stuck app with Apple’s built-in tools.**
   - Try closing the app normally first.
   - If the app does not respond, use **Force Quit** with **Option–Command–Esc**.
   - You can also open **Activity Monitor**, select the stuck app or process, and use **Quit** first.
   - Use **Force Quit** only if needed after saving other work; unsaved changes may be lost.
   - If you are unsure what to quit, restart the Mac instead of experimenting.

8. **Escalate workplace files early.**
   - Record the exact message, the file name, the location, and the time.
   - Follow your organization’s policy and contact IT or the appropriate internal support route if the file is sensitive, shared widely, or still locked after these steps.

## What can wait
- You do not need to rebuild the document from scratch now.
- You do not need to reinstall Office as a first step.
- You do not need to merge or delete competing versions until you have preserved a safe copy and confirmed which version is current.

## Important reassurance
A locked-document message often has a fixable cause, such as another open session, a background app, a file that did not close properly, or a sync delay. Preserving a separate copy first gives you room to troubleshoot without making the shared original harder to recover.

## Scope note
These are first steps to preserve your work and regain access. If the lock remains on shared storage, later decisions about permissions, version history, checkout settings, or server-side sessions may need help from IT.

## Important note
This is general information, not professional IT, legal, compliance, or other professional advice. For a sensitive or workplace file, follow your organization’s policy and involve IT rather than making repeated changes to the only copy.
