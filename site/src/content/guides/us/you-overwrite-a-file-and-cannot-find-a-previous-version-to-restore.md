# What to do if… you overwrite a file and cannot find a previous version to restore (USA)

## Short answer
Stop saving, stop editing, and pause syncing for that file’s folder if it is in a cloud-synced location. Then look for a recoverable copy in cloud version history, app version history, local backup, or a managed work or school backup before more changes overwrite your options.

## Do not do these things
- Don’t keep editing and re-saving the file to see what is left.
- Don’t restore an older version over the current file until you have made a separate safe copy of the current file.
- Don’t move, rename, or reorganize lots of files in the same synced folder right now.
- Don’t run cleanup utilities, reinstall apps, or try multiple recovery tools at random.
- Don’t keep sync running while you are trying to find the recoverable version.

## What to do now
1. Stop the overwrite from spreading.
   - Close the app that overwrote the file.
   - If the file is in a synced folder, pause OneDrive, SharePoint sync, Google Drive for desktop, or Dropbox sync if you can.
   - If you cannot quickly find the pause option, temporarily disconnect that device from the internet.

2. Preserve the current overwritten file without changing it.
   - Copy the current file to a separate location outside the synced folder, such as your desktop, a new folder, or an external drive.
   - Give the copy a clear name such as `overwritten-current-copy`.
   - Do not open and re-save that copy yet.

3. Check the cloud deleted-files area, even if the file was overwritten rather than deleted.
   - In OneDrive or SharePoint, check Recycle bin on the web.
   - In Google Drive, check Trash.
   - In Dropbox, check Deleted files.
   - If you find a likely copy, download or restore it to a new safe location rather than replacing your only current copy.

4. Check cloud version history from the web version of the service.
   - In OneDrive or SharePoint, select the file and open Version history.
   - In Google Drive, use Manage versions for uploaded files. For Google Docs, Sheets, or Slides, open File, then Version history, then See version history.
   - In Dropbox, open Version history for the file.
   - Where possible, download the older version, restore a copy, or copy its contents into a new file instead of immediately overwriting the current file.

5. If many OneDrive files changed, check whether OneDrive restore is available.
   - On the OneDrive website, look for Restore your OneDrive.
   - This option is generally for Microsoft 365 subscribers and may be limited to recent changes.
   - Do not use a whole-OneDrive restore until you understand what else it will roll back.

6. If you are on Windows, check whether a previous version or backup exists.
   - In File Explorer, right-click the folder that contained the file and choose Restore previous versions.
   - If a previous version appears, open or copy from it first instead of replacing the current folder.
   - If File History was turned on, use File History to restore the file or folder to a new safe location where possible.

7. If you are on macOS, check app versions and Time Machine.
   - In apps that support it, open the file and choose File, then Revert To, then Browse All Versions.
   - If you use Time Machine, enter Time Machine and look for the earlier version of the file or folder.
   - Restore to a copy or separate location where possible.

8. Check the app’s own version or recovery menu if it has one.
   - In Microsoft 365 apps, version history usually depends on the file being stored in OneDrive or SharePoint.
   - In other apps, look for a version, history, revert, recovery, or duplicate option.
   - Save any recovered file outside the synced folder until you are sure it is the right version.

9. If this is a work or school file, contact IT or the file owner quickly.
   - Say: “A key file was overwritten, sync is paused, and I need a restore from version history, backup, or snapshot.”
   - Include the file name, folder location, approximate overwrite time, device name, and cloud service.
   - If the file contains sensitive personal, medical, financial, legal, or regulated information, use your organization’s security or incident-reporting route rather than sending the file through chat or email.

## What can wait
- You do not need to choose a paid recovery tool right now.
- You do not need to repair or rewrite the document before checking version history and backups.
- You do not need to decide who is at fault right now.
- You can investigate why the previous-version option was missing after the file is stabilized.

## Important reassurance
An empty “Previous versions” menu does not always mean the old file is gone. The recoverable copy may be in a cloud version history, app version history, Time Machine, File History, or an organization-managed backup.

## Scope note
These are first steps to prevent further loss and check the main recovery routes. Later decisions may need help from IT support, a cloud-service admin, a data-recovery professional, or a compliance specialist.

## Important note
This is general information, not professional IT, legal, compliance, financial, therapeutic, or medical advice. Recovery depends on the storage service, device, app, backup settings, account type, and what happened after the overwrite.
