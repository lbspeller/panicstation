# What to do if… you overwrite a file and cannot find a previous version to restore (UK)

## Short answer
Stop editing the file and pause syncing if you can do that safely, then check cloud, app, and backup version history before more changes overwrite the last recoverable copy.

## Do not do these things
- Don’t keep opening and re-saving the file to check it; each save may replace more recoverable data.
- Don’t restore a version over the current file until you have made a safe copy, or until you can restore to another location.
- Don’t rename, move, or reorganise lots of files in the same folder yet, especially in a synced folder.
- Don’t run cleaners, optimisers, reinstall apps, or try several recovery tools at random.
- Don’t leave sync running if you can pause it safely before checking recovery routes.

## What to do now
1. Freeze the situation:
   - Close the app that overwrote the file.
   - If the file is in OneDrive, SharePoint, Google Drive, Dropbox, or another synced folder, pause syncing or briefly disconnect from the internet before making more changes.
   - Do not edit the file while you check recovery options.

2. Make a safe copy of the overwritten file:
   - Copy the current file to a different location, such as your Desktop, an external drive, or a folder outside the sync folder.
   - Give the copy a clear name such as `overwritten-current-copy`.
   - Do not open or re-save that copy yet.

3. Check the cloud bin or deleted-files area:
   - In OneDrive or SharePoint, check the Recycle bin on the web.
   - In Google Drive, check Bin.
   - In Dropbox, check Deleted files.
   - If you find the old file, download it or restore it to a safe place rather than replacing the current file straight away.

4. Check cloud version history:
   - In OneDrive or SharePoint, open the file on the web, choose Version history, and look for the last good version.
   - If many OneDrive files were affected, check Restore your OneDrive on the OneDrive website if it is available to your account; Microsoft says this is for Microsoft 365 subscribers and covers actions within the last 30 days.
   - In Google Drive, use Manage versions for uploaded files. For Google Docs, Sheets, or Slides, use File, then Version history.
   - In Dropbox, use Version history from the file’s menu. The time window depends on your Dropbox plan.
   - Where possible, download, copy, or restore the old version to a new location before replacing anything.

5. If Windows “Restore previous versions” is empty, check File History:
   - In File Explorer, right-click the folder that contained the file, then choose Restore previous versions.
   - If the file was at the top level of a drive, right-click the drive instead.
   - If File History shows a previous version, use Open to inspect it first. Use Restore to... if available so you do not overwrite the current file.

6. If you are on a Mac, check document versions and Time Machine:
   - In apps that support it, use File, Revert To, then Browse All Versions.
   - If you use Time Machine, open the folder that contained the file and restore the earlier version from the backup.
   - Prefer making a new copy first, rather than replacing the only current file.

7. Check the app’s own recovery options:
   - In Microsoft Word, Excel, or PowerPoint, check Version History inside the app for cloud files.
   - Check the app’s recovery or AutoRecover options if the file was local and the app offers them.
   - Save any recovered content immediately under a new filename in a new location.

8. If this is a work or school file, contact IT early:
   - Tell them: “A key file was overwritten; syncing is paused; I have made a safe copy; can you check backups, snapshots, SharePoint, Teams, or OneDrive version history?”
   - Ask them not to overwrite the current file while they investigate.
   - If the file is in SharePoint, Teams, or OneDrive for work or school, administrators may have recovery options you cannot see.

9. If the file contains personal data and this may be a personal data breach:
   - Start a simple incident log: what happened, when you noticed, what file was affected, and what you have done to contain it.
   - Follow your organisation’s route, such as your manager, IT security, Data Protection Officer, or information governance contact.
   - If the incident meets the reporting threshold, the UK ICO says a personal data breach must be reported without undue delay and within 72 hours of becoming aware.

## What can wait
- You do not need to decide now whether to pay for recovery software or a specialist service.
- You do not need to fix or rewrite the document yet.
- You do not need to work out why “previous versions” was missing until after you have checked cloud history, app history, File History, Time Machine, and any work or school backups.
- You do not need to explain the mistake to everyone before stabilising the file.

## Important reassurance
Overwriting a file is a common mistake, especially with sync tools, autosave, and similar filenames. “Previous versions” being missing in one place does not mean every recovery route is gone.

## Scope note
These are first steps to stop further loss and check the most common restore routes. Later decisions may need help from IT, an administrator, a data recovery specialist, or a data protection professional.

## Important note
This is general information, not legal, technical, financial, therapeutic, or professional advice. Recovery depends on where the file was stored, whether versioning or backups were enabled, and how much has changed or synced since the overwrite.
